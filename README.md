# MA_MachineLearning

This repository contains the implementation for extracting, processing, training, and evaluating neural networks for object detection in images as part of a master's thesis project. The core of the repository is a modular object detection pipeline that processes input images through a series of well-defined stages: **encoder**, **patch sampler**, **patch extractor**, and **classifier**. This structured approach enables efficient and accurate detection and classification of objects such as balls, penalty marks, and intersections in image data.

---

## What Does This Repository Do?

The primary goal of this repository is to provide a complete workflow for object detection in images, specifically tailored for robotic soccer (B-Human) applications. The workflow includes:

- **Data Extraction and Preparation**: Tools to extract and preprocess image data from B-Human logs, including handling raw image files and associated metadata.
- **Object Detection Pipeline**: A CNN-based pipeline that detects and classifies objects in images using feature extraction, patch sampling, inverse perspective mapping, and classification.
- **Training and Evaluation**: Scripts and utilities to train models on annotated datasets, monitor training progress, and evaluate model performance using various metrics.
- **Visualization and Analysis**: Tools to visualize dataset statistics, training progress, and evaluation results, aiding in the interpretation and debugging of the models.

  - If you are a B-Human member, generate a GitLab access token with **API permissions**.
  - Export the token as an environment variable:
    ```bash
    export UV_INDEX_PYBH_PASSWORD=<your-access-token>
    ```

3. **Install Dependencies**
  Use `uv sync` to synchronize the project dependencies as specified in the `pyproject.toml` file. This command will create a virtual environment and install all required packages.

## Structure of the Object Detection Algorithm

The object detection pipeline in this repository is designed with modularity and clarity in mind. It consists of the following key components:

1. **Encoder**
  The encoder processes the input image and generates feature maps that highlight regions of interest. It outputs interest maps and offset predictions for each object category, which are used in subsequent stages of the pipeline.
  - The encoder architectures are defined in [`src/training/encoder_architectures.py`](src/training/encoder_architectures.py).
  - The encoder can be configured with parameters such as input dimensions, number of channels, and whether to use batch normalization.
2. **Patch Sampler**
The patch sampler selects grid cells from the encoder's output based on interest scores and distance masks. This ensures that only the most relevant regions of the image are processed further, improving efficiency.
  - The patch sampler is implemented in [`src/util/layers.py`](src/util/layers.py) as the `PatchSampler` class.
  - It uses the interest scores and distance masks generated by the encoder to determine the most promising patches.
3. **Patch Extractor**
  The patch extractor takes the selected grid cells and extracts the corresponding image patches for classification. This process involves:
  - **Coordinate Transformation**: Converting the encoder's output coordinates (grid cell indices and offsets) into image coordinates using the camera matrix and intrinsic camera parameters (e.g., focal length, optical center).
  - **Inverse Perspective Mapping**: Projecting the 2D image coordinates to the ground plane using the camera pose (roll, pitch, height) and intrinsics (cx, cy, fx, fy). This allows the calculation of the 3D position of each patch relative to the camera.
  - **Distance Calculation**: For each predicted offset, the distance to the camera is computed using the camera matrix and intrinsic parameters. This distance is used to filter patches based on physical plausibility (e.g., objects cannot be behind the camera or too far away).
  - **Patch Normalization**: Extracted patches are normalized and resized to a fixed resolution for consistent input to the classifier.
  - The patch extractor is implemented in [`src/util/layers.py`](src/util/layers.py) as the `PatchExtractor` class.
  - It supports configurable patch sizes and object dimensions, allowing flexibility in handling different types of objects.
4. **Classifier**
  Each object category (e.g., ball, penalty mark, intersection) has its own dedicated classifier, which is trained to classify patches extracted for that specific category. The classifiers are designed as follows:
  - **Binary Classifiers**: For categories like **ball** and **penalty mark**, a binary classifier determines whether the patch contains the object.
  - **Multi-Class Classifier**: For the **intersection** category, a multi-class classifier distinguishes between different types of intersections.
  - **Offset Prediction**: In addition to classification, each classifier can predict an offset to correct the predicted offset from the encoder, improving spatial accuracy.
  - The classifier architectures are defined in [`src/training/classifier_architectures.py`](src/training/classifier_architectures.py).
  - The classifiers can be configured with parameters such as the number of classes, patch size, and whether to predict offsets.

The entire pipeline is integrated into the `FullModel` class in [`src/training/models.py`](src/training/models.py), which orchestrates the flow of data through the encoder, patch sampler, patch extractor, and classifier.

## Content

The repository is organized into directories, each containing scripts and utilities related to a specific aspect of the project.

### Dataset

The `src/dataset` directory contains scripts for handling all aspects of the dataset, from extraction to preparation for training.

- **Data Extraction from Logs**  
The script [`src/dataset/import_bhlogs.py`](src/dataset/import_bhlogs.py) parses and extracts data from B-Human logs, including reading raw image files and associated metadata.
- **Data Annotation**  
The [`src/dataset/labeling.py`](src/dataset/labeling.py) script provides tools for annotating extracted data, including functionalities for labeling objects in images.
- **Dataset Visualization**  
The [`src/dataset/statistics.py`](src/dataset/statistics.py) script visualizes dataset statistics, such as the distribution of object classes and image dimensions. The [`src/plotting`](src/plotting) directory contains utilities for creating various plots.
- **Dataset Generation**  
The [`src/dataset/split_dataset.py`](src/dataset/split_dataset.py) script splits the dataset into training, validation, and test sets. The [`src/dataset/save_dataset.py`](src/dataset/save_dataset.py) script provides functionalities to save and load datasets.

### Training

The `src/training` directory contains all scripts and utilities related to training the models.

- **Training Pipeline**  
The main training script, [`src/training/train.py`](src/training/train.py), trains the full model (encoder + classifier) or individual components. It supports configurable training parameters such as batch size, learning rate, and number of epochs. Training can be monitored using TensorBoard.
- **TensorBoard Integration**  
Training progress, including metrics such as loss and accuracy, can be logged and visualized using TensorBoard. To start TensorBoard, use:
  ```bash
  tensorboard --logdir=logs/
  ```
- **Encoder and Classifier Architectures**  
The encoder and classifier architectures are defined in:
  - [`src/training/encoder_architectures.py`](src/training/encoder_architectures.py)
  - [`src/training/classifier_architectures.py`](src/training/classifier_architectures.py)  
  These files contain implementations of various architectures, including custom layers such as `IresBlock` and `Normalization`.

### Evaluation

The `src/evaluation` directory contains scripts for evaluating the performance of the trained models.

- **Thresholdless Evaluation**  
The [`src/evaluation/thresholdless_metrics.py`](src/evaluation/thresholdless_metrics.py) script evaluates the encoder and classifier without relying on fixed thresholds.
- **Visualization of Results**  
The [`src/plotting`](src/plotting) directory and the [`src/evaluation/latex_tables.ipynb`](src/evaluation/latex_tables.ipynb) notebook contain utilities for visualizing evaluation results, such as ROC curves and confusion matrices.
- **Evaluation with Thresholds**  
The [`src/evaluation/evaluate.py`](src/evaluation/evaluate.py) script allows users to browse through the dataset and run the model on individual images to visualize prediction results directly on the images. This is useful for qualitative assessment, debugging, and demonstrating the model's performance.
